# Evaluación de Conocimientos: GitHub y Markdown
 
## 1. Conceptos Generales
**¿Qué es un archivo Markdown y para qué sirve?**
> Un archivo Markdown es un archivo de texto que se usa para dar formato de manera sencilla.
> Sirve para poder escribir un texto de manera clara sin necesidad de usar otras herramientas.
 
**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**
 
--- *El propósito es presentar información sobre el tema en un repositorio en GitHub.*

## 3. Flujo de Trabajo en GitHub Desktop
**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**
* **Commit:** (Registra y guarda los cambios que se realicen agregando un mensaje que  lo descriva )
* **Push:** (Permite subir los commits al servidor para que otras personas puedan comentar o editar el programa)
 
**¿Para qué sirve el botón "Fetch Origin"?**
> Sirve para ver los cambios realizados por otras personas sin que se apliquen directamente al repositorio.
